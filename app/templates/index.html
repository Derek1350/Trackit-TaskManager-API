Flask API Documentation

Welcome to the Flask API documentation! This API is designed to manage tasks with role-based access control (RBAC), leveraging Flask, PostgreSQL, Redis, Celery, and Docker for containerization. Below, you'll find details on how to interact with the API endpoints.

1. User Registration

URL: /auth/register

Method: POST

Body: application/json

Description: Register a new user by providing a username, email, and password. Upon successful registration, the user will receive a JWT token for authentication.

Request Body: 

        {
            "username": "user123",
            "email": "user123@example.com",
            "password": "password123"
        }

Response: A JWT token will be returned for authentication.

Example:

        POST /auth/register HTTP/1.1
        Content-Type: application/json

        {
            "username": "user123",
            "email": "user123@example.com",
            "password": "password123"
        }

2. User Login

URL: /auth/login

Method: POST

Body: application/json

Description: Log in to the application by providing the registered email and password. Upon successful login, the user will receive a JWT token for authentication.

Request Body: 

        {
            "email": "user123@example.com",
            "password": "password123"
        }

Response: A JWT token will be returned for authentication.

Example:

        POST /auth/login HTTP/1.1
        Content-Type: application/json

        {
            "email": "user123@example.com",
            "password": "password123"
        }

3. Get All Tasks

URL: /tasks

Method: GET

Authorization: Bearer JWT Token (Admin or User role)

Description: Retrieve a list of tasks. Admin can view all tasks, while a user can only view their own tasks.

Response: A list of tasks, or an empty list if no tasks exist.

Example:

        GET /tasks HTTP/1.1
        Authorization: Bearer

4. Create a Task

URL: /tasks

Method: POST

Authorization: Bearer JWT Token (User role)

Description: Create a new task. The task will be assigned to the user making the request.

Request Body: 

        {
            "title": "Task Title",
            "description": "Task Description"
        }

Response: A success message with the created task details.

Example:

        POST /tasks HTTP/1.1
        Authorization: Bearer 
        Content-Type: application/json
        
        {
            "title": "New Task",
            "description": "This is a new task description"
        }

5. Update a Task

URL: /tasks/{id}

Method: PUT

Authorization: Bearer JWT Token (User role, or Admin)

Description: Update a task by ID. Only the user who created the task or an Admin can update it.

Request Body: 

        {
            "title": "Updated Task Title",
            "description": "Updated task description"
        }

Response: A success message with the updated task details.

Example:

        PUT /tasks/1 HTTP/1.1
        Authorization: Bearer 
        Content-Type: application/json
        
        {
            "title": "Updated Task",
            "description": "Updated description for task"
        }

6. Delete a Task

URL: /tasks/{id}

Method: DELETE

Authorization: Bearer JWT Token (Admin only)

Description: Delete a task by ID. Only Admin can delete a task.

Response: A success message confirming task deletion.

Example:

        DELETE /tasks/1 HTTP/1.1
        Authorization: Bearer

Technology Stack

  • Flask: Python web framework for building RESTful APIs.
  • PostgreSQL: Relational database for storing task and user data. (Dockerized)
  • Redis: In-memory data store used for caching and message brokering (Celery).
  • Celery: Distributed task queue for handling background tasks asynchronously.
  • JWT: JSON Web Token for authentication and role-based access control (RBAC).
  • Docker: Used to containerize the application, database, and services for easy deployment.
  • Docker Compose: Defines and runs multi-container Docker applications for the entire stack.

How to Run Locally

To run this Flask API locally, make sure you have Docker and Docker Compose installed.

        # Clone the repository
        git clone 

        # Navigate to the project directory
        cd 

        # Build and run the Docker containers
        docker-compose up --build

The application will be running on http://localhost:5000.

API Authentication

To interact with the API endpoints, you need to authenticate using JWT tokens. Here's how:

  1. Register a user and obtain a JWT token via the /auth/register endpoint.
  2. Log in to get a JWT token via the /auth/login endpoint.
  3. Include the token in the Authorization header of your requests as a Bearer token.
